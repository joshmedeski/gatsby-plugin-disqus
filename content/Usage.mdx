---
title: "Usage"
---

Add the plugin to your `gatsby-config.js` file with your <a href='https://help.disqus.com/installation/whats-a-shortname' target='_blank' rel='noopener noreferrer'>Disqus shortname</a>

```javascript
// gatsby-config.js
module.exports = {
  plugins: [
    {
      resolve: `gatsby-plugin-disqus`,
      options: {
        shortname: `your-disqus-shortname`
      }
    },
  ]
}
```
The plugin can be used as shown in this brief example:  

```jsx
import { Disqus, CommentCount } from 'gatsby-plugin-disqus'

const PostTemplate = () => {
  let disqusConfig = {
    url: `${config.siteUrl+location.pathname}`,
    identifier: post.id,
    title: post.title,
  }
  return (
    <>
      <h1>{post.title}</h1>
      <CommentCount config={disqusConfig} placeholder={'...'} />
      /* Post Contents */
      <Disqus config={disqusConfig} />
    </>
  )
}

export default PostTemplate
```  
While providing a `url`, `identifier`, and `title` are optional, these attributes will prevent threads from being lost in the case that the domain changes or the post is renamed.
